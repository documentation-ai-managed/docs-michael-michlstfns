## Overview

Michael / Michlstfns provides a powerful documentation space where you organize, manage, and share all your project documentation. You create structured pages using MDX, leverage rich components for interactive content, and maintain a professional knowledge base effortlessly. Designed for developers, teams, and projects of any size, it streamlines your workflow from initial setup to advanced publishing.

## Quick Start

Get up and running in minutes. Follow these steps to create your first documentation page.

<Steps>
  <Step title="Sign Up" icon="user-plus">
    Visit `https://michael.michlstfns.com` and create your account. No credit card needed.
  </Step>
  <Step title="Create Workspace" icon="folder-plus">
    Set up a new documentation space for your project.
  </Step>
  <Step title="Add First Page" icon="file-plus">
    Use the MDX editor to draft your `introduction.mdx` page.
  </Step>
  <Step title="Publish" icon="globe">
    Preview and publish your docs live.
  </Step>
</Steps>

<CodeGroup tabs="npm,yarn">
  ```bash
  npm install mdx-documentation-kit
  ```
  ```bash
  yarn add mdx-documentation-kit
  ```
</CodeGroup>

## Brand Guidelines

Follow these guidelines to maintain consistency across your documentation.

<Callout kind="info">
  Primary brand color: `#3B82F6`. Use it for accents, buttons, and links.
</Callout>

```css
:root {
  --brand-primary: #3B82F6;
  --brand-text: #1F2937;
}

.button {
  background-color: var(--brand-primary);
  color: white;
}
```

<Expandable title="Advanced Customization" default-open="false">
  Override CSS variables in your theme file:
  
  ```javascript
  export const theme = {
    colors: {
      primary: '#3B82F6'
    }
  };
  ```
</Expandable>
